# 设计模式

- 一年的工程师不怎么学，3年+的一定要学的，大项目宝典
- 设计模式是leetcode 之外，顶级大厂代码能力的测试题
  抽象 代码不是写出来的，AIGC写代码（推荐），代码的设计
  是设计模式
  高手不写（怎么），设计代码（架构师）


- 码商+情商 = 面向对象
  对象在JS里面很轻松 
  {}-->对象字面量
  对象 = 属性 + 方法
  代码的运行的过程是工作流的处理过程，要把流程搞完整

- 实现同样方法的对象（设计模式就叫做实现了同样的接口），就可以互换使用，
  面向对象-> 面向接口编程

- 定义对象的方法时，例如：sender:function(){} 在es6中可简化为sender(){}
